# UI 心理学约束

## 目标

安的书房后续所有功能和界面，都要先满足一个前提：

- 对中文小白友好；
- 低认知负担；
- 桌面与 Android 共用同一套信息结构；
- 不靠堆按钮、堆卡片、堆动效制造“丰富感”。

网页只作为预览和分发入口。真正的产品约束，优先服务 Windows 桌面和 Android。

## 约束来源

这份约束综合以下权威来源整理：

- Apple Human Interface Guidelines
- Apple Accessibility / Cognitive
- Apple Loading / Buttons
- Android Developers Material Design / Material Components
- Nielsen Norman Group 的可用性启发式，尤其是 `Recognition Rather Than Recall`

## 硬规则

### 1. 每屏只保留一个主任务

- 每个页面只允许 1 个主动作。
- 同一视图里，强强调按钮最多 1 到 2 个。
- 如果一个页面需要用户同时理解“阅读、安装、聊天、筛选、下载、跳转”，就必须拆层。

落地标准：

- 首页第一屏只回答“这是谁的书房、我从哪里开始”。
- 安装页第一屏只回答“我该装哪一个端”。
- 工坊页第一屏只回答“我该点哪一个项目开始”。

### 2. 先识别，再记忆

- 不要求用户记住路径、术语、模块关系后再操作。
- 所有关键入口都必须显性可见。
- 路由命名只用公开统一名称：`首页 / 藏馆 / 谱系 / 工坊 / 风信 / 手记 / 年谱 / 书房 / 安装 / 小安对话`。

落地标准：

- 小安入口必须同时有文字，不只靠头像或图标。
- 卡片标题必须直接说明用途，不写内部黑话。
- 新手路径优先使用“先看什么、接着做什么”的句式。

### 3. 复杂流程必须渐进展开

- 多步骤任务不要一次塞满一个屏幕。
- 长说明优先拆成“判断 / 依据 / 下一步 / 提醒”。
- 一个屏幕只让用户处理一类决策。

落地标准：

- 详情页先给摘要，再给步骤，再给失败点。
- 安装流程按“预览 -> 桌面 / Android 选择 -> 对应更新方式”展开。
- 小安回答最多给 3 条下一步。

### 4. 不能出现空白等待

- 任何远程加载都必须给骨架屏、占位或进度反馈。
- 不允许 blank page、突然闪出内容、长时间静止无解释。

落地标准：

- 列表页先显示骨架，再替换真实内容。
- 图片先显示占位，再替换。
- 小安思考中必须给可理解的加载提示。

### 5. 动效只服务方向感，不服务炫技

- 禁止高频、反复、无信息价值的动效。
- 过渡动画只用来说明“页面切换了”“面板打开了”“内容已加载”。
- 需要兼容减少动态效果偏好。

落地标准：

- 不做持续漂浮、强视差、强弹簧、闪烁动画。
- 弹窗、抽屉、对话框使用短动画和明确起止。
- 视觉焦点变化优先用淡入淡出，不靠复杂位移动画。

### 6. 信息密度高，但阅读阻力低

- 内容可以丰富，但同屏元素不能互相争抢。
- 每个卡片只呈现一个判断、一句用途、一个下一步。
- 列表页不允许每张卡都展开成长文。

落地标准：

- 工坊卡片默认跳转详情，不在列表里堆长正文。
- 藏馆卡片优先写“适合谁 / 解决什么 / 下一步做什么”。
- 风信卡片优先写“发生了什么 / 为什么改变判断 / 读者怎么行动”。

### 7. 平台习惯不能反着来

- Android 和桌面都尽量用平台熟悉模式。
- 桌面强调稳定导航、窗口阅读、应用内更新。
- Android 强调触控、单手、分步、轻量入口。

落地标准：

- Android 不复用桌面 updater。
- 桌面和 Android 共享内容与 API，但不强行共享交互细节。
- 桌面按钮尺寸、Android 触控热区、返回方式分别按端优化。

## 页面级检查清单

每次做新页面或改版，都要先过这一遍：

1. 第一屏是不是只承担一个主任务？
2. 主按钮是否超过两个？
3. 是否要求用户先记住术语再操作？
4. 是否把复杂流程一次性压给用户？
5. 加载时是否会出现空白或不确定状态？
6. 动效是否可以删除而不影响理解？如果可以，默认删。
7. 手机端单手点击是否容易误触？
8. 桌面端是否保留长期阅读的稳定感？
9. 文案是否像写给中国小白，而不是写给工程师？
10. 这个页面是否让用户更清楚下一步，而不是更累？

## 当前执行顺序

接下来两端界面按这个顺序收：

1. 小安入口与对话体验
2. 安装页的双端决策路径
3. 首页第一屏的身份与起步路径
4. 工坊列表与详情页的低负担跳转
5. 藏馆与谱系的卡片信息密度控制

## 结论

安的书房不能靠“内容很多”显得厉害。
真正的标准是：读者不慌，知道先点哪里，做完一步以后还能继续走下去。
