# 🏢 MetaGPT SOP 框架内化: 软件公司隐喻与标准化流程

> **核心理念**: **“将 SOP 注入 Agent 大脑”**。通过模拟软件公司的完整岗位（PM, Architect, Engineer），实现从需求到代码的全自动生产。

---

## 1. 🏗️ 核心架构

### 1.1 软件公司隐喻
- **Roles**: 产品经理、架构师、项目经理、工程师。
- **Artifacts**: PRD、系统设计图、单元测试、源代码。

### 1.2 标准作业程序 (SOP)
- 强制 Agent 遵循严格的顺序：需求评审 -> 架构设计 -> 代码开发 -> 测试。
- **价值**: 通过流程的确定性补偿模型输出的不确定性。

---

## 2. 🧩 知识共享 (Message Pool)

- **核心机制**: 所有 Agent 共享一个消息池，Agent 会根据自己的角色订阅相关消息。
- **环境隔离**: Agent 在其特定的“环境”中工作，互不干扰。

---

## 🚀 accio 演进启示
- **标准化 SOP**: 在 `99 - Agent指南与规范` 中完善任务流水线，规定每一步必须产生的“产出物 (Artifacts)”。
- **岗位化封装**: 强化目前 Pro-Team 的岗位边界。
