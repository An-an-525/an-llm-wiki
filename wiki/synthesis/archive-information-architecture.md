# Archive Information Architecture

The archive should become useful as an information system before it becomes polished as a portfolio. This page defines the public information backbone.

## Core Tables

- [[wiki/synthesis/project-evidence-matrix]] records project themes and public readiness.
- [[wiki/synthesis/capability-evidence-matrix]] connects capabilities to evidence and repeatable workflows.
- [[wiki/synthesis/public-growth-timeline-framework]] defines how growth and timeline information can be made public without exposing private records.
- [[wiki/synthesis/information-review-queue]] keeps unresolved information decisions visible.

## Information Priority

The highest-value information is not necessarily the newest file. Priority comes from:

- repeated evidence across multiple source groups;
- ability to explain a project or capability publicly;
- low privacy risk;
- usefulness for a future reader;
- connection to existing wiki pages.

## Public Shape

Each public information page should say what is known, what is inferred, what remains private, and what needs review. This is stricter than a normal portfolio page, but it keeps the archive trustworthy.
