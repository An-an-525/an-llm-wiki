# 从单次提问到 Agent 工作流

这个阶段的变化，是从“问 AI 一个问题”变成“组织一组 Agent 完成一件事”。

## 发生了什么

安开始把 Agent 当成工作流里的参与者，而不是神秘的万能按钮。每个 Agent 都需要范围、交付物和验收门槛；每个技能都要说明什么时候用、怎么用、怎样判断做完。

## 公开经验

Agent 协作要想可靠，至少要有四件事：

- 一个明确负责人；
- 一个清楚的写入范围；
- 一个可以交接的结果；
- 一个真正跑过的验证。

心理层：从“让 AI 替我做”转向“我来设计 AI 怎么协作”，人的位置会变得更稳。焦虑感会下降，因为你不再把结果完全交给黑箱。

社会层：多人协作里需要分工，人与 Agent 的协作也一样。没有边界的智能，很容易变成没有责任的热闹。

哲学层：工具越强，越需要人给它结构。自由不是让所有东西同时发生，而是知道哪一步该由谁承担。
