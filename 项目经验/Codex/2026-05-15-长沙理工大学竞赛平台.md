# 长沙理工大学竞赛平台 Codex 项目经验

## 项目目标

为长沙理工大学云塘校区土木工程与环境学院竞赛工作构建一个竞赛信息、在线报名、管理员后台、数据导出和 AI 答疑平台。

## 当前仓库

- 本地路径：`C:\Users\fkl26\Documents\Codex\2026-05-12\https-qingyixia111-github-io-csust-competition`
- GitHub 仓库：`https://github.com/An-an-525/csust-civil-competition-platform`
- 公网页面：`https://an-an-525.github.io/csust-civil-competition-platform/`

## 关键决策

- GitHub Pages 只作为静态前端展示，不承担真实数据库、密钥保存和服务端 AI。
- 真实业务能力由 `render-backend` 提供，包括登录、报名、导出、附件上传、AI 调用和数据存储。
- AI 密钥必须只放后端环境变量，不能写进前端或 GitHub Pages。
- 当前数据层使用 JSON 文件，适合演示和小规模部署；长期上线建议 PostgreSQL。
- 竞赛种子数据只保留官方可核验来源，未确认字段显示“待通知”或“以当届通知为准”。

## 已完成能力

- 学生端：首页、竞赛列表、搜索、详情、小白说明、团队报名。
- 管理端：登录、竞赛创建编辑、发布、开关报名、报名查看、状态修改、备注、CSV / Excel 导出、附件上传。
- AI：后端调用 MiniMax M2.7 快速版，优先读取平台数据和官方来源，缺少依据时不能编造。
- 文档：README、部署说明、项目计划、待办清单、领导验收简报、小白配置指南。
- PDF：领导验收简报 PDF、小白配置指南 PDF。

## 部署状态

- GitHub Pages 前端已可访问。
- 后端曾通过 Cloudflare Tunnel 公网连通，但该地址属于临时演示，不适合作为长期生产地址。
- 正式上线需要固定服务器、域名、HTTPS 和持久化数据目录。

## 配置要点

- 本地运行：`npm install`，`npm run dev`。
- 默认管理员：`admin` / `admin123`，正式上线必须改。
- AI：配置 `LLM_API_KEY`、`LLM_BASE_URL=https://api.minimaxi.com/v1`、`LLM_MODEL=MiniMax-M2.7-highspeed`。
- 生产：必须设置强 `JWT_SECRET` 和强管理员密码。
- GitHub Pages 连后端：仓库变量设置 `CSUST_API_BASE=https://正式后端域名`。

## 验证结果

- `npm run build` 通过。
- `npm run package` 通过。
- 公网前端返回 200。
- 后端 `/api/health` 曾验证正常，显示 `llmConfigured=true`、`competitionCount=12`。
- 管理员登录接口曾验证可返回 token。

## 已知风险

- 当前未购买服务器，公网后端不能长期保证稳定。
- JSON 文件数据层不适合多人长期高并发使用。
- GitHub Pages 演示模式的 localStorage 不能当正式数据源。
- 默认管理员密码如果不改，会有安全风险。
- MiniMax API Key 不能提交进仓库。

## 下一步动作

1. 把本地未提交文档、PDF 和配置指南提交并推送到 GitHub。
2. 购买服务器后部署 `render-backend`。
3. 配置正式域名、HTTPS、`JWT_SECRET`、`SEED_ADMIN_PASSWORD`、`LLM_API_KEY`。
4. 将数据层迁移到 PostgreSQL。
5. 在 GitHub 仓库变量设置 `CSUST_API_BASE` 指向正式后端。
6. 完整验收：首页、详情、报名、后台登录、导出、AI 问答、附件上传。

## 可复用经验

- 面向领导验收时，不要只说“页面做好了”，要拆成展示层、业务层、运行层。
- AI 问答产品要明确证据边界：平台数据优先，官方来源补充，缺失信息不编造。
- GitHub Pages + 后端 API 是低成本演示方案，但正式使用必须引入持久后端和数据库。
- 项目交付最好同时提供 README、部署说明、小白配置指南和验收简报 PDF。
