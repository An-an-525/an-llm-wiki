# 安的书房跨端发布工作流

这份文档用于工程协作，不是公网展示内容。

## 当前目标

先让网页、Windows 桌面版和 Android APK 共用同一套公开内容，并保证读者能稳定打开、阅读、问小安和获取新版。

## 当前形态

| 端 | 当前状态 | 交付物 | 更新方式 |
| --- | --- | --- | --- |
| Web | 已可用 | 公开网页 / PWA | 刷新后读取最新静态资源和公开数据 |
| Windows | 已有 0.1.1 安装包 | NSIS 安装包 | 读取 Tauri updater 清单，应用内更新 |
| Android | 已有 arm64 内测 APK | APK | 读取 release.json，用户手动下载新版 APK |

## 关键边界

- 安装包只带公开前端，不带密钥、token、SSH、raw、private-wiki 或本地协作日志。
- 小安模型服务走服务端，不把访问材料写入前端仓库或安装包。
- 内容更新和版本更新分开：内容可以更频繁，安装包只在壳体、能力、修复需要时更新。
- Android 现阶段不做静默更新，必须由用户确认安装。

## 发布顺序

1. 更新公开内容与前端。
2. 跑构建、内容质量、隐私、wiki 检查。
3. 准备 Web 静态资源。
4. 准备 Windows 安装包和 updater 清单。
5. 准备 Android APK 与 release.json。
6. 上传到服务器公开目录。
7. 用网页、Windows、Android 分别检查：
   - 首页能打开；
   - 藏馆、工坊、谱系能进入详情；
   - 小安能回答公开问题；
   - 更新页能读到远程清单；
   - 更新失败时保留旧内容。

## 小鲸鱼交接点

小鲸鱼只负责本地事实采集和证据摘要。它的结果进入内容编译流程，不直接进入安装包或公网数据。

当小鲸鱼交付资料时，Codex 只接收：

- public-candidate 的事实摘要；
- 可公开的项目、工具、路线和失败记录；
- 风险标记；
- 需要安确认的问题。

不接收：

- 密钥明文；
- 服务器登录细节；
- 本地绝对路径；
- 原始聊天全文；
- 未脱敏截图和私密材料。

## 下一步

- 给 Android 做正式签名策略。
- 给服务器目录定最终域名和路径，避免影响现有中转站。
- 给 release.json 增加 Web 内容版本字段。
- 给小安服务增加公开上下文版本号，方便排查“内容已更新但回答还旧”的问题。
