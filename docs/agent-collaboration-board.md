# 安的书房协作任务板入口

这份文档用于在 Obsidian 内定位协作状态。具体长期任务板和日志放在外部协作日志库。

## 看板列

| 列 | 含义 |
| --- | --- |
| Inbox | 刚收到的想法、问题和材料 |
| 澄清中 | 需要安确认目标、边界或事实 |
| 可执行 | 已写明可读范围、可写范围和验收标准 |
| 进行中 | 正在由小鲸鱼、小安、Codex 或审阅子代理处理 |
| 待验收 | 已产出，需要构建、内容质量、隐私或浏览器检查 |
| 已归档 | 已完成并写回记录 |

## 任务卡字段

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| id | 是 | 稳定任务编号 |
| 标题 | 是 | 一句话说明 |
| 来源 | 是 | 来自用户、raw、前端问题或审阅 |
| 负责人 | 是 | 小鲸鱼、小安、Codex、审阅子代理 |
| 状态 | 是 | 对应看板列 |
| 可写范围 | 是 | 明确允许修改哪些文件 |
| 输入材料 | 是 | 用摘要描述，不贴敏感原文 |
| 输出文件 | 是 | 预计产生或修改的位置 |
| 验收标准 | 是 | 通过条件 |
| 安全级别 | 是 | public-safe、private-summary、raw-only、secret-risk、local-path-risk |
| 更新时间 | 是 | 最后一次变更时间 |

## 小鲸鱼交接箱

小鲸鱼正在做本地事实采集时，把结果写成事实卡，不写成公网最终文案。每张事实卡先回答五个问题：

1. 这条事实来自哪类材料；
2. 它能帮助读者理解安的哪段路；
3. 它能不能公开，为什么；
4. 如果能公开，应该进入首页、藏馆、谱系、工坊、风信、手记、年谱还是小安；
5. 如果不能公开，能否改写成方法、提醒或失败记录。

### 接收标准

- 只接收事实、摘要、风险和公开候选，不接收密钥、账号、完整原始聊天和本机路径。
- 没有来源说明的内容只能进入“待复核”，不能直接进前端。
- 能帮小白复刻的内容优先于工具名清单。
- 每次接收后必须写入外部协作日志，再决定是否进入公开数据。

## 前端晋升门槛

事实卡进入读者前端前，必须补成资料包。资料包不是一段介绍，而是一页能被读者打开、判断和照做的内容。

| 必填 | 最低要求 |
| --- | --- |
| 读者 | 写清第一次做的人、已会一点的人、想深入的人分别怎么看 |
| 用途 | 说清它帮读者解决什么具体问题 |
| 操作 | 至少给出一个当天能完成的最小动作 |
| 失败 | 至少写一个真实可能踩到的坑 |
| 检查 | 告诉读者怎样知道自己做对了 |
| 边界 | 写清哪些内容不公开，为什么 |
| 来源 | 用公开来源或脱敏来源摘要支撑，不写本机路径 |

拒收：

- 只有工具名，没有使用场景；
- 只有观点，没有步骤；
- 只有情绪，没有事实；
- 只有后台链接，读者打不开；
- 需要读者理解本地黑曜石或 raw 才能读懂；
- 带密钥、账号、路径、服务器细节、原始聊天或未复核个人事实。
