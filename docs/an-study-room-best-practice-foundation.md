# 安的个人书房最佳实践底层框架

本文件说明安的个人书房为什么这样设计。它不是随意拼出来的前端，也不是把黑曜石内容直接搬到网页上，而是把公开知识库、中文学习体验、个人成长叙事、AI 工具复刻和安全发布放在同一套标准里。

## 1. 总判断

当前项目必须同时满足四层逻辑：

1. **知识库逻辑**：原始材料不直接公开，公开页面是经过编译、链接、复核的知识成果。
2. **学习逻辑**：小白不是来背技术名词的，而是要靠 AI 做成一件具体事。
3. **叙事逻辑**：这是安的个人书房，要有人、时间、转折、判断和边界。
4. **工程逻辑**：前端、数据、后端、小安、App 化和发布流程都要可验证。

如果一个内容只满足其中一层，它不能直接进入核心展示。

## 2. 外部最佳实践来源

| 来源 | 采用的底层逻辑 | 在本项目中的落地 |
|---|---|---|
| Karpathy LLM Wiki | raw sources、wiki、schema 三层分离；wiki 是持续积累的编译成果 | `_raw/local-only` 不进公开层，`wiki/` 是公开改写层，`AGENTS.md/CLAUDE.md` 约束 Agent |
| OpenAI Codex Best Practices + `openai/codex` | 用 `AGENTS.md` 存长期规则；把重复工作做成 skill；稳定后再自动化；短而准、可验证 | `AGENTS.md` 不再堆长提示；skill 只做单一工作；自动化只调度稳定流程；技能要有 smoke test 和触发样例 |
| Obsidian Properties | 用 YAML/frontmatter、tags、aliases、dates 建立结构化元数据 | 所有公开页面必须有 frontmatter、来源、状态、日期和公开安全字段 |
| Diátaxis | 教程、操作指南、参考、解释分别服务不同读者需求 | 藏馆、谱系、工坊、手记不混写；复刻包必须有教程和操作步骤 |
| NN/g 信息架构与可用性 | 用用户语言和用户心智模型组织导航，不用内部黑话 | 藏馆改为“做前端、做后端、用工具、搭智能体”等行动分类 |
| 认知负荷理论 | 新手一次处理不了太多新信息，需要例子、步骤、减少无关信息 | 卡片只做判断入口，详情页给完整复刻；不在卡片里展开长文 |
| React / Web 前端实践 | 组件职责清楚、数据集中、状态可预测、交互语义正确 | 前端只读生成数据，导航用链接，动作才用按钮 |
| PWA / Service Worker 实践 | 缓存和更新策略要明确，安装后也能获得新数据 | App 化优先缓存外壳，公开数据走运行时刷新 |
| OWASP 安全实践 | 访问材料不能进入前端、公开仓库和日志；需要轮换、最小暴露和扫描 | 小安认证配置只在本地忽略环境，公开层只写方法和边界 |

## 3. 心理学层

本项目默认读者有三类心理负担：

- **信息焦虑**：资料太多，不知道先看什么。
- **能力焦虑**：看到别人做了很多 AI 项目，以为自己无法开始。
- **判断疲劳**：工具、模型、平台太多，不知道哪些值得学。

因此内容必须降低进入门槛：

- 每张卡片先讲“这能帮你做什么”。
- 每个详情页要有“最小复刻版本”。
- 每个项目要写失败点，让读者知道不是自己笨，而是系统本来复杂。
- 每个模块都要给下一步行动，不让读者读完停在情绪里。

心理层不能写成自我诊断，也不能写成鸡汤。它必须绑定真实操作，例如：材料太多导致混乱、任务太长导致上下文丢失、工具太多导致选择困难。

## 4. 社会学层

AI 学习不是一个人在真空里完成。它总在平台、工具、学校、社区、市场和协作方式里发生。

本项目的社会学层要解释：

- 为什么 Kimi、Codex、Claude Code、Coze、GitHub、Obsidian 等工具会改变工作方式。
- 为什么公开展示需要比私人笔记更强的边界和来源。
- 为什么读者需要看到复刻路线，而不是只看到成品。
- 为什么小安不是炫技入口，而是读者和资料库之间的解释者。

社会学层不能写成宏大空话。每一条都要回到具体系统：前端、后端、知识库、工具链、发布流程、学校项目、个人学习路径。

## 5. 哲学层

本项目采用“实践智慧”的方向：知识不是收藏本身，知识要能指导行动、承担边界、接受修正。

因此每个高质量内容都要回答：

- 这件事教会了什么判断？
- 哪些东西不能做，为什么？
- 什么是当前阶段的最小完成？
- 如果未来发现错了，应该怎样更新？

哲学层要克制。它不负责拔高，它负责把经验放回人的选择、限制、时间和责任里。

## 6. 模块底层逻辑

| 模块 | 底层逻辑 | 内容形态 |
|---|---|---|
| 首页 | 第一眼建立“安的个人书房” | 人、书房、小安、入口、精选路线 |
| 藏馆 | 读者想靠 AI 做什么 | 行动分类卡、资料卡、工具卡、提示词卡 |
| 谱系 | 新手需要路线 | 阶段、前置条件、交付物、验收 |
| 风信 | 信息要变成判断 | 趋势、认知变化、下一步行动 |
| 工坊 | 项目要能复刻 | 背景、工具、操作、失败、提醒 |
| 手记 | 经历要能沉淀 | 事件、反思、方法修正 |
| 年谱 | 成长要有时间线 | 真实日期、阶段变化、相关页面 |
| 小安 | 对话要有边界 | 中文、克制、客观、只基于公开资料和用户输入 |

## 7. 内容准入底线

一个页面要进入公开展示，最低必须满足：

- 标题和摘要是中文，能被小白理解。
- 有真实来源或明确的公开安全来源说明。
- 有下一步行动。
- 对工坊项目，必须有真实操作、复刻步骤、失败点、安的提醒。
- 对风信，必须有趋势、判断变化、行动建议。
- 对谱系，必须有阶段和验收方式。
- 对小安，必须有边界，不输出访问材料、本机路径和私密事实。

不满足这些条件，宁愿不上架。
