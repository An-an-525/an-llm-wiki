# 校内基金路演项目说明

本稿按路演口径整理，给代讲人直接使用。  
原则是：不把所有零散尝试都堆进去，而是按“已经交付使用、已完成阶段成果、正在推进、下一阶段计划”来讲。

不建议在正式路演里同时混用“已经做了的 / 做完的 / 在做的”这类重复状态，因为听起来不专业，也容易让评审听不出项目成熟度。

---

## 一句话定义

我们不是只做单个 AI 小工具，而是在做一套**面向中文用户的 AI 学习与成长资料平台**：  
把项目实践、工具使用、Agent 工作流、模型实验和个人成长经验，整理成**可展示、可复盘、可复刻**的系统。

---

## 我们现在主要在做什么

可以概括为三条线：

1. **做实际项目**：例如竞赛系统前端、Agent 工作流、模型实验闭环。  
2. **做平台化整理**：把分散的项目、工具和方法沉淀成知识库和展示前端。  
3. **做可复刻的方法体系**：不是只展示结果，而是把“怎么做、哪里会失败、下一步怎么走”整理出来，方便后续展示、学习和扩展。

---

## 一、已经交付使用的

### 1. 竞赛系统前端项目

- 已经完成上线，可通过 GitHub Pages 访问。
- 已完成静态落地页、多模块页面和基础跳转结构。
- 这是目前最明确的“已经交付、已经可使用”的项目。
- 它证明我们已经具备把想法快速落地为真实可访问产品的能力。

路演讲法：

> 我们不是停留在想法阶段，已经有项目完成上线并投入实际展示和使用。竞赛系统前端是我们已交付的代表案例，说明我们具备从需求理解、页面实现到部署上线的完整执行能力。

---

## 二、已完成的阶段成果

这一部分不是“已经全部商业化完成”，而是“底层能力和阶段性成果已经做出来了”。

### 1. 个人资料库平台总体架构

- 已完成平台的总体定义和边界设计。
- 已把系统拆成原始材料层、私有整理层、公开知识层、前端数据层。
- 已明确哪些内容能公开、哪些必须保留在本地。
- 这解决了“资料很多但无法安全展示”的核心问题。

### 2. Karpathy 风格知识库重构

- 已完成公开 wiki 的重构基础。
- 已建立知识编译逻辑，不再把原始资料直接堆到展示层。
- 已建立内容来源、结构校验、隐私边界和公开清单的基本规则。
- 这意味着我们已经不是简单记笔记，而是在做一套可维护的知识系统。

### 3. 路演与公开展示的内容标准

- 已完成平台内容标准、模块标准和公开边界标准。
- 已明确首页、藏馆、谱系、工坊、手记、年谱、小安各自的职责。
- 已明确什么样的内容才允许进入公开展示。
- 这让后续内容扩展不会越来越乱。

### 4. Codex / Agent 技能治理基础

- 已完成本地技能体系和官方最佳实践的第一轮对齐。
- 已明确 `AGENTS.md`、`skill`、`automation` 的边界。
- 已开始把重复经验整理成可复用方法，而不是靠临时聊天推进。
- 这为后续多人协作和多 Agent 协作打基础。

路演讲法：

> 我们已经完成的不只是几个页面，而是一套平台底层：包括知识结构、公开边界、内容标准、工程规则和协作方式。这些基础设施决定了项目能不能长期扩展，而不是只做一次展示。

---

## 三、正在推进的

### 1. 个人资料库展示前端

- 正在持续建设中。
- 已有前端骨架、公开数据读取方式、模块划分和 PWA 方向。
- 当前重点是补高质量内容，而不是只堆界面。
- 目标是把它做成“安的个人书房”式的公开学习展柜。

### 2. Coze 风格 Agent 搭建器研究

- 正在推进中。
- 当前不是盲目复刻完整平台，而是聚焦最小可验证 Agent 工作流。
- 已经形成公开安全的架构研究页和最小闭环思路。
- 下一步是把节点图、运行记录和示例流程做得更可演示。

### 3. 小安本地模型学习闭环

- 这条线当前处于整理与继续推进之间。
- 已经形成了目标定义、数据检查、小样本试跑、指标比较、复盘闭环的方法。
- 当前重点不是直接讲“训了一个模型”，而是讲“我们建立了一套模型实验方法”。
- 这更适合路演，因为它体现的是能力，而不是一次性结果。

### 4. 学习路线与可复刻资料包

- 正在推进中。
- 目标是把项目、工具和方法整理成小白能看懂的中文复刻路径。
- 不是只展示成果，而是把经验变成别人能照着走一遍的资料包。
- 这会成为平台后续最有教育价值和展示价值的部分。

路演讲法：

> 我们现在不是只有一个项目，而是在同时推进前端展示、Agent 工作流、模型实验和学习资料平台化。它们看起来分散，但实际上都在指向同一个目标：把 AI 实践沉淀成可展示、可复刻、可持续迭代的系统。

---

## 四、下一阶段计划

### 1. 把平台从“可看”推进到“可学习”

- 继续补 3 到 5 个高质量项目页。
- 补 2 到 3 条完整学习路径。
- 补 5 到 10 个工具或 Agent 卡片。
- 让小白第一次进入时就知道从哪里开始。

### 2. 做稳定的公开数据出口

- 继续完善 `site-data` 和后续只读 API 的数据契约。
- 让前端、未来 App 和小安都只基于公开安全数据工作。
- 为后续服务器部署和功能扩展打基础。

### 3. 推进小安与 Agent 展示能力

- 继续补齐小安的公开问答边界、资料来源和可解释性。
- 把 Agent 工作流从研究页逐步推进到可演示原型。
- 目标不是做炫技机器人，而是做一个可靠、克制、可解释的数字助手入口。

### 4. 建立长期展示与存档能力

- 每一个项目不仅要做出来，还要沉淀成以后可展示、可申报、可复用的正式资产。
- 这样后面无论是比赛、基金、展示、合作还是更大规模的路演，都有现成材料可用。

---

## 五、为什么值得支持

### 1. 我们已经不是空想阶段

- 已经有真实交付项目。
- 已经有知识库平台和展示平台底层。
- 已经有清楚的工程边界、内容边界和隐私边界。

### 2. 我们做的不是一次性作品

- 不是单个网页，也不是单个实验。
- 而是一套能持续沉淀、持续扩展、持续展示的系统。

### 3. 这个方向具备校园创新价值

- 能服务 AI 学习展示
- 能服务项目沉淀和成果表达
- 能服务后续教学、比赛、展示、合作和路演

### 4. 资金的作用会很明确

支持资金不是为了“从零开始试试看”，而是为了加速已经有基础的系统继续成型，例如：

- 前端和交互完善
- 后端接口与部署
- 小安与 Agent 原型完善
- 内容资料包补全
- 演示和展示能力增强

> 现在正在推进的是个人资料库展示前端、Coze 风格 Agent 搭建器研究、小安本地模型学习闭环，以及面向小白的复刻资料包。

---

## 七、路演时不要这样讲

- 不要把所有零散项目平铺直叙。
- 不要只讲“我们用了很多 AI 工具”。
- 不要把未完成研究讲成已经成熟产品。
- 不要把知识库讲成普通笔记系统。

应该讲的是：

- 我们已经交付过什么；
- 我们已经沉淀出什么底层能力；
- 我们现在在推进什么；
- 为什么这件事值得继续支持。
